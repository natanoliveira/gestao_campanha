## Jornada do Usuário por Persona

**Onboarding (Primeira Vez)**
1. Recebe convite para plataforma
2. Cria conta e ativa MFA
3. Realiza tour guiado
4. Configura organizações e estrutura
5. Define políticas de permissão
6. Importa usuários existentes
7. Configura integrações (se necessário)
8. Executa primeira auditoria
9. Completa onboarding

**Operação Regular**
1. Login diário
2. Revisa dashboard executivo
3. Verifica alertas e notificações críticas
4. Analisa relatórios de conformidade
5. Gerencia aprovações pendentes
6. Atribui permissões a novos usuários
7. Revisa logs de auditoria (quando solicitado)

## Níveis de Acesso Detalhados

### Nível 1: Sem Acesso
- Usuário não consegue visualizar o módulo
- Acesso negado em tentativa de acesso direto

### Nível 2: Leitura (R)
- Visualizar informações
- Gerar relatórios
- Exportar dados
- Sem permissão de criar, editar ou deletar

### Nível 3: Leitura + Comentários (R+)
- Tudo de Nível 2
- Adicionar comentários
- Reagir a publicações
- Deixar feedback

### Nível 4: Edição (U)
- Tudo de Nível 2
- Editar dados
- Atualizar status
- Sem permissão de criar ou deletar

### Nível 5: CRUD Completo
- Criar novo registro
- Ler/visualizar
- Atualizar/editar
- Deletar (soft delete)

## Considerações Especiais

### Acesso ao Portal Público
- Doadores e Visitantes acessam portal público
- Dados limitados a informações publicadas
- Sem acesso a dados sensíveis (financeiro privado, membros, etc.)
- Acesso anônimo permitido

### Acesso por Projeto
- Gestores acessam apenas projetos atribuídos
- Administrador acessa todos os projetos
- Membros acessam apenas projetos atribuídos
- Auditores acessam todos os projetos

### Soft Delete
- Usuários deletados não são removidos do log
- Auditoria continua disponível
- Restauração possível por administrador

### Auditoria de Mudanças
- Toda alteração é registrada
- Identificação de quem fez a mudança
- Timestamp preciso
- Antes e depois dos dados
- Auditores têm acesso irrestrito
